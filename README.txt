Консольное приложение, демонстрирующее казуальный пример работы банковских счетов и операций с ними
на основе MVC-паттерна

Базовый класс Account создается не напрямую, а через класс-строитель (AccBuilder);
Класс Transaction представляет собой некоторую операцию, которая при успешном исполнении записывается;

Модель (Bank) содержит основные операции: создание счета, пополнение, снятие
    (в идеале, для полноты, не хватает перевода со счета на счет)
    а также хранит список транзакций и счетов;
Презентер под собой имеет меню;
Контроллер - промежуточное звено между моделью и презентером, однако некоторую часть функционала из презентера
    можно перенести, для удобочитаемости;

В приложении реализованы обработки ошибок, таких как:
-при внесении отрицательного баланса при создании счета;
-при пополнении/снятии отрицательной суммы;
-при снятии средств по объему превышающих баланс счета (нестандартный Exception);
-при выборе несуществующего счета
